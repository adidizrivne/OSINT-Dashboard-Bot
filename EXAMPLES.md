# 📖 Приклади Використання OSINT Dashboard Bot

## 🚀 Швидкий Старт

### 1. Встановлення та Запуск

```bash
# Клонування або завантаження файлів
git clone <repo-url>
cd osint-bot

# Автоматичне налаштування
python3 setup.py

# Або ручне встановлення залежностей
pip install -r requirements.txt

# Запуск бота
python3 osint_bot.py
```

### 2. У Telegram

```
1. Знайди свого бота в пошуку (за username)
2. Напиши /start
3. Обери функцію з меню
4. Введи свої дані
5. Отримай результати
```

## 📚 Практичні Приклади

### Приклад 1: Дослідження Username

**Сценарій:** Ти знайшов цікавий профіль на GitHub та хочеш дізнатися, де ще активна ця людина.

**Кроки в Telegram:**
```
/start
[Натисни: 🔍 Sherlock (Username)]
Введи: john_doe
```

**Результат:**
```

• GitHub: https://github.com/john_doe
• Reddit: https://reddit.com/user/john_doe
• Twitter: https://twitter.com/john_doe
• Stack Overflow: https://stackoverflow.com/users/john_doe
... (ще 8 сайтів)
```

**Як це використати:**
- ✅ Дослідження розробника перед наймом
- ✅ Пошук контактів людини
- ✅ Перевірка чи справжній профіль

### Приклад 2: Геолокація IP

**Сценарій:** Ти одержав залогований спам з чогось сумнівного. Хочеш дізнатися звідкіля.

**Кроки:**
```
/start
[Натисни: 🌍 GeoIP (IP Адреса)]
Введи: 192.168.1.1
```

**Результат:**
```
🌍 GeoIP Інформація:

🌐 Країна: United States
📍 Регіон: California
🏙️ Місто: San Francisco
📐 Координати: 37.7749, -122.4194
🔗 ISP: Google LLC
🏢 Організація: Google
📡 AS: AS15169
```

**Як це використати:**
- ✅ Перевірка IP серверу перед з'єднанням
- ✅ Визначення джерела атаки
- ✅ Перевірка VPN/Proxy
- ⚠️ Точність: ±10км (залежить від мережі)

### Приклад 3: WHOIS Дослідження Домену

**Сценарій:** Ти знайшов підозрілий сайт та хочеш дізнатися його деталі.

**Кроки:**
```
/start
[Натисни: 🏢 WHOIS (Домен)]
Введи: example-suspicious.com
```

**Результат:**
```
🏢 WHOIS Інформація:

🌐 Домен: example-suspicious.com
📋 Реєстратор: GoDaddy
📅 Створено: 2024-03-15
🔄 Оновлено: 2025-12-20
⏰ Закінчується: 2026-03-15
🔗 NS: ns1.example.com, ns2.example.com
```

**Як це використати:**
- ✅ Перевірка давності домену (свіжі домени = 🚩 підозріло)
- ✅ Перевірка дати закінчення
- ✅ Дослідження реєстратора
- ✅ Виявлення фішингу (домени, схожі на оригінальні)

### Приклад 4: Перевірка Email у витоках

**Сценарій:** Ти отримав лист про витік даних. Хочеш перевірити твій email.

**Кроки:**
```
/start
[Натисни: 🔓 HaveIBeenPwned (Email)]
Введи: myemail@example.com
```

**Результат A (Скомпрометований):**
```
🔓 HaveIBeenPwned Результат:

⚠️ УВАГА! Цей email був скомпрометований!
📊 Кількість витоків: 3
📝 Витоки: LinkedIn, Adobe, Equifax
```

**Результат B (Безпечно):**
```
🔓 HaveIBeenPwned Результат:

✅ Добра новина! Цей email не був знайдений у витоках.
```

**Що робити якщо знайдено:**
1. ✅ Змінити пароль на цьому сайті
2. ✅ Змінити пароль на емейлі (якщо втік email+пароль)
3. ✅ Увімкнути двофакторну аутентифікацію
4. ✅ Перевірити інші облікові записи з таким же паролем

### Приклад 5: Email Search - Аналіз репутації

**Сценарій:** Ти отримав фішинговий лист від "support@bank.com" та хочеш перевірити чи це реально.

**Кроки:**
```
/start
[Натисни: 📧 Email Search]
Введи: support@fake-bank.com
```

**Результат:**
```
📧 Email Search Результат:

⭐ Репутація: 85 (1-100, вище = краще)
🚨 Підозріли: Так
📬 Доставляється: Ні
✔️ Формат валідний: Так
```

**Інтерпретація:**
- 🚩 "Репутація: низька" + "Підозріли: Так" = **Фішинг!**
- ✅ "Репутація: висока" + "Підозріли: Ні" = **Legitime**

## 🎯 Кейси Використання

### 1. **Безпека Облікових Записів**

Частотність: **Кожний місяць**

```
Процес:
1. Виконуй /start
2. Введи всі свої email адреси в HIBP
3. Якщо знайдено - змініть пароль
4. Повтори щомісяця
```

### 2. **Дослідження Безпеки Перед Наймом**

Частотність: **Кожна вакансія**

```
Кандидат: John Doe
1. Sherlock пошук: john_doe
2. WHOIS: його сайт/портфоліо
3. GeoIP: звідки він логується
4. HIBP: переконайся чи не скомпрометований
```

### 3. **Аналіз Фішингу**

Частотність: **При підозрі**

```
Сумнівний лист від "admin@bank.com":
1. GeoIP: яка IP адреса сервера
2. WHOIS: домен справжній?
3. Email Search: чи підозріла адреса?
4. HIBP: був цей email у витоках?
```

### 4. **Дослідження Конкурентів** (Законне)

Частотність: **Щотижня**

```
Для компанії competitor.com:
1. WHOIS: дата реєстрації, реєстратор
2. GeoIP: хостинг в якій країні?
3. Email Search: компанійні email адреси
4. Sherlock: пошук офіційних акаунтів
```

## 🐛 FAQ & Вирішення Проблем

**Q: Sherlock дуже довгий!**
A: Це нормально. Шерлок шукає на 300+ сайтах. Займає 2-5 хвилин.

**Q: GeoIP показує невірне місце**
A: Точність залежить від ISP. Часто буває відхилення в 10-50км.

**Q: Отримую помилку API**
A: Перевір інтернет з'єднання та ліміти API.

**Q: Email Search мені не показує детали?**
A: Цей email чистий. Нема інформації у базах.

**Q: Я запросив занадто багато запитів?**
A: API ліміти:
- IP-API: 45/хв
- HIBP: 1500/год
- Чекай або придбай ключ.
